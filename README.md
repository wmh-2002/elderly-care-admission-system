# 养老院入住管理系统需求规格说明书

## 1. 引言

### 1.1 编写目的

本文档旨在详细描述基于SpringBoot的养老院入住管理系统的功能需求、性能需求、设计约束等，为系统的设计、开发、测试和验收提供依据。

### 1.2 项目背景

随着我国人口老龄化程度不断加深，养老院作为重要的养老服务机构，其管理效率和服务质量面临着巨大挑战。传统的人工管理方式已无法满足现代化养老院的管理需求，亟需一套信息化管理系统来提升管理效率和服务水平。

### 1.3 定义

- **SpringBoot**：基于Spring框架的快速开发框架
    
- **MySQL**：开源关系型数据库管理系统
    
- **B/S架构**：浏览器/服务器架构模式
    

## 2. 系统概述

### 2.1 系统目标

开发一套基于SpringBoot的养老院入住管理系统，实现老人入住、护理、费用、员工等全方位的信息化管理，提高养老院的管理效率和服务质量。

### 2.2 系统特点

- 基于SpringBoot框架开发，具有良好的扩展性和维护性
    
- 采用B/S架构，用户通过浏览器即可访问系统
    
- 界面友好，操作简便
    
- 数据安全可靠，权限控制完善
    

## 3. 功能需求

### 3.1 系统用户角色

1. **系统管理员**：拥有系统所有权限
    
2. **院长**：查看各类统计报表，决策支持
    
3. **护理主管**：管理护理人员和护理记录
    
4. **护理员**：记录老人日常护理信息
    
5. **财务人员**：管理费用相关功能
    
6. **前台接待**：负责老人入住登记
    

### 3.2 功能模块划分

#### 3.2.1 老人档案管理

- **老人信息录入**：录入老人基本信息、联系方式、紧急联系人等
    
- **老人信息查询**：支持多条件组合查询老人信息
    
- **老人信息修改**：修改老人基本信息
    
- **老人信息导出**：导出老人信息到Excel
    
- **老人照片管理**：上传和管理老人照片
    

**老人信息字段**：

- 基本信息：姓名、性别、年龄、身份证号、出生日期、民族、籍贯
    
- 联系方式：联系电话、家庭地址
    
- 健康信息：血型、过敏史、既往病史、目前健康状况
    
- 入住信息：入住日期、房间号、床号、护理等级
    
- 费用信息：收费标准、缴费方式
    
- 紧急联系人：姓名、关系、联系电话
    

#### 3.2.2 入住管理

- **入住登记**：新老人入住信息登记
    
- **房间分配**：根据老人需求和房间状态分配房间
    
- **床位管理**：管理各房间的床位状态
    
- **入住协议**：生成和管理入住协议
    
- **退住处理**：处理老人退住相关手续
    

#### 3.2.3 护理管理

- **护理等级设置**：设置不同护理等级及标准
    
- **护理计划制定**：为每位老人制定个性化护理计划
    
- **护理记录**：记录日常护理情况
    
- **护理评估**：定期对老人进行护理评估
    
- **护理提醒**：护理任务提醒功能
    

**护理记录内容**：

- 生命体征：体温、脉搏、呼吸、血压
    
- 饮食情况：进食量、特殊饮食要求
    
- 排泄情况：大小便情况
    
- 睡眠情况：睡眠质量、时间
    
- 用药情况：用药名称、剂量、时间
    
- 特殊情况：异常症状、突发事件
    

#### 3.2.4 费用管理

- **费用标准设置**：设置不同护理等级的收费标准
    
- **费用计算**：根据入住时间和护理等级自动计算费用
    
- **费用缴纳**：记录老人缴费情况
    
- **费用查询**：查询老人费用缴纳历史
    
- **欠费提醒**：对欠费老人进行提醒
    
- **财务报表**：生成各类财务报表
    

**费用类型**：

- 住宿费：根据房间类型收费
    
- 护理费：根据护理等级收费
    
- 餐饮费：根据饮食标准收费
    
- 医疗费：医疗服务费用
    
- 其他费用：特殊服务费用
    

#### 3.2.5 员工管理

- **员工档案**：管理员工基本信息
    
- **排班管理**：安排员工工作时间
    
- **考勤管理**：记录员工考勤情况
    
- **工资管理**：计算和管理员工工资
    
- **权限管理**：设置员工系统操作权限
    

#### 3.2.6 访客管理

- **访客登记**：记录访客基本信息
    
- **访客查询**：查询访客记录
    
- **访客统计**：统计访客情况
    

#### 3.2.7 物资管理

- **物资入库**：记录物资入库信息
    
- **物资出库**：记录物资出库信息
    
- **库存查询**：查询当前库存情况
    
- **库存预警**：库存不足时自动预警
    

#### 3.2.8 系统管理

- **用户管理**：管理系统用户信息
    
- **角色管理**：设置不同角色权限
    
- **数据备份**：定期备份系统数据
    
- **系统日志**：记录系统操作日志
    
- **参数设置**：设置系统基本参数
    

### 3.3 统计报表功能

- **老人统计**：按年龄、性别、护理等级等维度统计
    
- **费用统计**：收入、支出统计分析
    
- **入住率统计**：房间入住率分析
    
- **员工统计**：员工结构分析
    
- **护理统计**：护理工作量统计
    

## 4. 非功能需求

### 4.1 性能需求

- **响应时间**：页面响应时间不超过3秒
    
- **并发用户**：支持50个并发用户同时在线
    
- **数据容量**：支持10年以上数据存储
    
- **系统可用性**：系统可用性达到99%
    

### 4.2 安全性需求

- **用户认证**：用户登录需要身份验证
    
- **权限控制**：不同角色拥有不同操作权限
    
- **数据加密**：敏感数据需要加密存储
    
- **操作日志**：记录所有重要操作日志
    
- **数据备份**：定期自动备份数据
    

### 4.3 可靠性需求

- **数据一致性**：保证数据的一致性和完整性
    
- **错误处理**：系统出现错误时能够友好提示
    
- **故障恢复**：系统故障后能够快速恢复
    

### 4.4 易用性需求

- **界面友好**：界面设计简洁美观
    
- **操作简便**：操作流程简单明了
    
- **帮助文档**：提供详细的帮助文档
    

### 4.5 可维护性需求

- **模块化设计**：系统采用模块化设计
    
- **代码规范**：代码编写遵循规范
    
- **文档完整**：提供完整的技术文档
    

## 5. 运行环境

### 5.1 硬件环境

- **服务器**：CPU 4核以上，内存8GB以上，硬盘500GB以上
    
- **客户端**：普通PC机，内存4GB以上
    

### 5.2 软件环境

- **操作系统**：Windows Server 2012及以上版本
    
- **数据库**：MySQL 8.0及以上版本
    
- **Web服务器**：Tomcat 9.0及以上版本
    
- **浏览器**：Chrome、Firefox、Edge等主流浏览器
    

### 5.3 开发环境

- **开发语言**：Java 8及以上版本
    
- **开发框架**：SpringBoot 2.5及以上版本
    
- **前端技术**：HTML5、CSS3、JavaScript、Vue.js
    
- **数据库**：MySQL 8.0

- **开发工具**：IntelliJ IDEA、Maven
